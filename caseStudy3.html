Overview

Fashion Stylist: Website/App

Project description

UX Case Study

Duration: 14th Febuary - 25th May

Product Overview

It delves into the designing and creation of both website as well as an app that provides recommendations about matching outfits to users.


Problem

Absence of dedicated fashion Stylist: Absence of a dedicated virtual AI fashion stylist for Bhutanese women's traditional clothing like tego, wonju, and kira.


Time Consuming: Current manual process of selecting appropriate color combinations for traditional garments is time-consuming and lacks technological solutions.


Goal

Develop both mobile app as well as website that helps users to select the best matching outfits for themselves.

Roles and Responsibilities

My Role

UI/UX Designer: Responsible for the overall design and user experience for Fashion Stylist System.

Responsibility

My responsibility involves conducting user research, designing wireframes, developing the app's features, testing and iterating, and implementing the final design and functionality.

User research: summary

For Fashion Stylist App/Website, I have conducted user research via surveys, interviews, and usability testing. Initially assuming customers prioritized convenience, I discovered they also seek engaging, visually rich experiences when selecting their prefered attires. This insight prompted me to focus on integrating AI models and interactive features to enhance user satisfaction.

Usability study

In my usability studies for the fashion stylist website/app, I conducted research to gather valuable insights from users. The insights revealed challenges and numerous parameters like Navigation efficiency, user engagement and User satisfaction.

Findings

  • Users noted they spent more time interacting with 3D images compared to standard images.
  • While users appreciated the quality of the content, they recommended enhancing readability by using larger fonts and more visual aids.
  • Feedback highlighted the need for more interactive elements to enhance user engagement.

Accessibility

Color Contrast

  • Ensured sufficient color contrast between text and background elements to improve readability and accommodate users with visual impairments.

    • Keyboard Navigation

  • Implemented keyboard accessibility for all interactive elements to enable users who rely on keyboard navigation or assistive technologies to navigate the app effectively.

    • Screen Reader Compatibility

  • Ensured proper labeling of interface elements and utilized ARIA attributes to optimize compatibility with screen readers, enhancing accessibility for visually impaired users
